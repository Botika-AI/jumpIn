<task type="checkpoint:human-action" gate="blocking">
  <what-built>
    Before running this checkpoint, complete these one-time Supabase setup steps (Claude cannot do these):
  </what-built>
  <how-to-verify>
    **Step 1: Create .env.local**
    Copy `.env.local.example` to `.env.local` and fill in your Supabase project values:
    - NEXT_PUBLIC_SUPABASE_URL → Supabase Dashboard → Project Settings → API → Project URL
    - NEXT_PUBLIC_SUPABASE_PUBLISHABLE_KEY → Supabase Dashboard → Project Settings → API → anon/publishable key

    **Step 2: Disable email confirmation**
    - Supabase Dashboard → Authentication → Providers → Email
    - Uncheck "Confirm email" → Save
    - (Without this, signUp returns session: null and the user lands stuck)

    **Step 3: Run schema.sql**
    - Supabase Dashboard → SQL Editor → New query
    - Paste the full contents of `supabase/schema.sql`
    - Click Run — expect "Success. No rows returned"
    - Confirm the `profiles` table appears in Table Editor

    Once all three steps are done, run `npm run dev` and proceed to the functional test in the next task.
  </how-to-verify>
  <resume-signal>Type "setup done" when .env.local is filled, email confirmation is disabled, and schema.sql has been run successfully.</resume-signal>
</task>

<task type="checkpoint:human-verify" gate="blocking">
  <what-built>
    Full Supabase authentication flow: registration with real user creation, login, session persistence, logout, error handling, and QR check-in writing to Supabase.
  </what-built>
  <how-to-verify>
    Run `npm run dev` and open http://localhost:3000

    **Test 1 — Confirm password mismatch:**
    1. Click "Crea un account" to go to registration
    2. Fill in any name, email, school, DOB
    3. Type "password1" in Password, "password2" in Conferma Password
    4. Click submit
    5. Expected: Italian error "Le password non coincidono." appears, form not submitted

    **Test 2 — Successful registration:**
    1. Fill all fields with a real email you own (or a test email)
    2. Make both password fields match (min 8 chars)
    3. Click "Registrati ora"
    4. Expected: Loading state → dashboard appears with your real name and profile data
    5. Verify in Supabase Dashboard → Table Editor → profiles: your row exists with correct data

    **Test 3 — Session persists:**
    1. While on dashboard, press F5 (hard refresh)
    2. Expected: Dashboard loads immediately (brief loading flash is acceptable), NOT the login form

    **Test 4 — QR check-in:**
    1. Click the QR scanner button on dashboard
    2. Scan any QR code (or allow camera and scan anything)
    3. Expected: Green success checkmark, last_checkin timestamp updates on dashboard
    4. Verify in Supabase Dashboard → profiles: last_checkin column updated for your row

    **Test 5 — Logout:**
    1. Click logout button
    2. Expected: Login form appears
    3. Open DevTools → Application → Cookies: Supabase session cookies should be cleared (or expired)

    **Test 6 — Login with real credentials:**
    1. Log in with the email/password from Test 2
    2. Expected: Dashboard with your profile, previous last_checkin timestamp visible

    **Test 7 — Wrong password:**
    1. Logout, then try logging in with correct email but wrong password
    2. Expected: Italian error "Credenziali non valide." on the login form

    **Test 8 — Duplicate email registration:**
    1. Go to registration, use the same email from Test 2
    2. Fill all fields, matching passwords
    3. Click submit
    4. Expected: Italian error "Email già registrata. Accedi invece."
  </how-to-verify>
  <resume-signal>Type "approved" if all 8 tests pass. If any fail, describe which test number and what happened instead of the expected behavior.</resume-signal>
</task>
