<task type="checkpoint:human-action">
  <name>Task 1: Set up Google credentials and QR values in .env.local</name>
  <what-to-do>
Before testing the Sheets integration, the following must be added to `.env.local` (create from `.env.local.example` if it does not exist):

1. **Create a Google Service Account** (if not already done):
   - Google Cloud Console → IAM & Admin → Service Accounts → Create Service Account
   - Grant no roles (or Viewer — access is controlled by Sheets sharing)
   - Keys tab → Add Key → JSON → download the JSON file

2. **Share your Google Spreadsheet** with the service account:
   - Open the JSON key file, find `"client_email"` value
   - Open your Google Sheet → Share → paste that email → Editor → Send

3. **Add to `.env.local`**:
   ```
   GOOGLE_SERVICE_ACCOUNT_KEY='<paste entire contents of JSON key file here>'
   GOOGLE_SPREADSHEET_ID=<your spreadsheet ID from the URL>
   ENTRANCE_QR_VALUE=<exact string content of entrance QR code>
   EXIT_QR_VALUE=<exact string content of exit QR code>
   ```
   Note: Wrap the JSON value in single quotes. Do not add spaces around `=`.

4. Restart the dev server: `npm run dev`

If you do not yet have real QR codes, you can test with placeholder strings (e.g., `ENTRANCE_QR_VALUE=test-entrada`). You can use a QR code generator to create a test QR that encodes that exact string.
  </what-to-do>
  <resume-signal>Type "ready" when .env.local is updated and dev server is running</resume-signal>
</task>

  <how-to-verify>
Run `npm run dev` (if not already running) and open http://localhost:3000.

**Test 1 — Entrance check-in:**
1. Log in with a test account
2. Tap/click "Effettua Check-in" to open QR scanner
3. Scan the entrance QR code (or manually test via the API if no camera):
   ```bash
   # Find a logged-in user's session cookie and test directly:
   curl -X POST http://localhost:3000/api/checkin \
     -H "Content-Type: application/json" \
     -d '{"nome":"Test","cognome":"User","email":"test@example.com","scuola":"Liceo Test","dataOra":"17/03/2026, 14:30:00","decodedText":"<your ENTRANCE_QR_VALUE>"}'
   ```
4. Verify: Response `{"ok":true}` and a new row appears in Google Sheets with Tipo=`Entrata`
5. Check that the row has correct Italian headers in row 1 (Nome, Cognome, Email, Scuola, Data e Ora, Tipo)

**Test 2 — Exit check-in:**
1. Scan the exit QR code (or use curl with EXIT_QR_VALUE)
2. Verify: New row in Sheets with Tipo=`Uscita`

**Test 3 — Full UI flow (if camera available):**
1. Open the dashboard, tap camera button, scan entrance QR
2. Verify: "Check-in Confermato!" success state appears in Dashboard
3. Verify: Supabase profile last_checkin timestamp updated
4. Verify: New row in Sheets with correct data

**Test 4 — Error handling (optional):**
1. Temporarily set `GOOGLE_SPREADSHEET_ID` to an invalid value in .env.local, restart dev server
2. Scan a QR code
3. Verify: Yellow Italian error banner appears: "Check-in registrato. Errore di sincronizzazione con il registro."
4. Verify: Supabase last_checkin was still updated (check Supabase dashboard)
5. Restore correct GOOGLE_SPREADSHEET_ID

**Test 5 — Header auto-init:**
1. If Sheets was empty before Test 1, row 1 should now contain: Nome | Cognome | Email | Scuola | Data e Ora | Tipo
