**Fix 1 — Unmount race guard:**
Add a `cancelled` boolean at the top of the useEffect, set it to `true` in the cleanup function, and check it before calling `start()`:
```ts
let cancelled = false;
// inside the .then() callback, before scannerRef.current = new Html5Qrcode(...)
if (cancelled) return;
```
In the cleanup:
```ts
return () => {
  cancelled = true;
  scannerRef.current?.stop().catch(console.error);
};
```

**Fix 2 — clientWidth guard with requestAnimationFrame:**
After creating the Html5Qrcode instance and before calling `.start()`, check if the container has a non-zero width. If zero, defer with requestAnimationFrame:
```ts
const startScanner = () => {
  if (cancelled) return;
  const container = document.getElementById('qr-reader');
  if (!container || container.clientWidth === 0) {
    requestAnimationFrame(startScanner);
    return;
  }
  scannerRef.current!.start(...).catch(handleError);
};
startScanner();
```

**Fix 3 — Expanded .catch() error detection:**
Replace the existing `.catch()` block (and the identical per-frame error handler) with a shared `handleError` function that covers all known failure modes and always shows an Italian error message for unrecognized errors:
```ts
const handleError = (err: unknown) => {
  const msg = String(err);
  if (
    msg.includes('NotAllowedError') ||
    msg.includes('NotReadableError') ||
    msg.includes('NotFoundError') ||
    msg.includes('OverconstrainedError') ||
    msg.includes('Camera streaming not supported') ||
    msg.includes('insecure context') ||
    msg.includes('getUserMedia') ||
    msg.includes('https')
  ) {
    setCameraError('Impossibile accedere alla fotocamera. Verifica i permessi del browser.');
  } else if (msg.length > 0 && !msg.includes('No MultiFormat Readers')) {
    // Generic fallback — not a per-frame "no QR found" message
    setCameraError('Errore fotocamera. Ricarica la pagina e riprova.');
  }
};
```
Use `handleError` in both the per-frame error callback and the `.catch()`.
