export type AuthState = 'login' | 'register' | 'dashboard' | 'loading';

**components/AuthController.tsx:**

1. Remove the `readSession` function entirely — it is no longer needed.

2. Add a `useIsomorphicLayoutEffect` helper at the top of the file (before the component), since Next.js static export has no server runtime but SSR safety is still required:
```ts
import { useState, useEffect, useLayoutEffect, useRef } from 'react';
const useIsomorphicLayoutEffect = typeof window !== 'undefined' ? useLayoutEffect : useEffect;
```

3. Change the useState initializers to unconditionally start at 'loading' / null:
```ts
const [authState, setAuthState] = useState<AuthState>('loading');
const [user, setUser] = useState<UserProfile | null>(null);
```

4. Add a useIsomorphicLayoutEffect immediately after the useState declarations that reads localStorage and sets the real auth state. This fires synchronously before paint on the client:
```ts
useIsomorphicLayoutEffect(() => {
  try {
    const saved = localStorage.getItem('jumpin_user');
    if (saved) {
      const parsed = JSON.parse(saved) as UserProfile;
      setUser(parsed);
      setAuthState('dashboard');
    } else {
      setAuthState('login');
    }
  } catch {
    localStorage.removeItem('jumpin_user');
    setAuthState('login');
  }
}, []);
```

5. Add a render guard at the top of the return logic, before the existing `if (authState === 'login')` block:
```ts
if (authState === 'loading') {
  return null;
}
```

6. Remove the now-unused `readSession` import from the file (it was a local function, not an import — delete the entire function definition).

Do NOT change handleLogin, handleRegister, handleLogout, handleCheckIn, or any other logic. Only the initialization and the render guard change.
