- Props: onScan (decodedText: string) => void, onClose: () => void
- useEffect dep array BUG in old code: uses [onScan] which causes re-initialization. New code MUST use [] (empty)
- Use onScanRef pattern to avoid stale closure:
  ```typescript
  const onScanRef = useRef(onScan);
  useEffect(() => { onScanRef.current = onScan; }, [onScan]);
  // Scanner useEffect with []:
  useEffect(() => {
    scannerRef.current = new Html5QrcodeScanner("qr-reader", { fps: 10, qrbox: { width: 250, height: 250 } }, false);
    scannerRef.current.render(
      (decodedText) => { onScanRef.current(decodedText); scannerRef.current?.clear(); },
      () => {} // ignore frame errors
    );
    return () => { scannerRef.current?.clear().catch(console.error); };
  }, []); // empty — initialize once
  ```
- If Html5QrcodeScanner import causes SSR build error ("document is not defined"), use dynamic import inside useEffect:
  ```typescript
  useEffect(() => {
    import('html5-qrcode').then(({ Html5QrcodeScanner }) => {
      // initialize scanner here
    });
  }, []);
  ```
